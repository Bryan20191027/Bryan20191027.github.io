2020/11/9 JAVA学习笔记 接口Interface

一.接口

1.接口：

1.定义：接口就是多个类的公共规范，是一种引用数据类型，最重要的是其中的方法
2.定义格式：

public interface 接口名称{  /*方法体*/  }

3.注意事项：

  • 换成了关键字interface之后编译生成的字节码文件仍然是：.java–>.class
  • java7接口中可以包含的内容有：1.常量 2.抽象方法
  • java8额外包含有：3.默认方法 4.静态方法
  • java9额外包含有：5.私有方法

4.接口使用步骤：

1.接口不能直接使用，必须有一个“实现类”来实现该接口
实现类格式：

public class MyInterfaceImp implements MyInterfaceA,MyInterfaceB{
    //覆盖重写所有抽象方法
}

2.接口的实现类必须覆盖重写(实现)接口中所有的抽象方法

  • 去掉abstract关键字，加上方法大括号

3.创建实现类的对象，进行使用

  • new需要是实现类

2.接口注意事项：

（1）接口是没有静态代码块或者构造方法的
（2）一个类的直接父类是唯一的，但是一个类可以同时实现多个接口
（3）如果实现类所实现的多个接口当中，存在重复的抽象方法，那么只需要覆盖重写一次即可
（4）如果实现类没有覆盖重写所有接口当中的所有抽象方法，那么实现类就必须是一个抽象类
（5）如果实现类所实现的多个接口当中，存在重复的默认方法，那么实现类一定要对冲突的默认方法进行重写
（6）一个类如果直接父类当中的方法，和接口当中的默认方法产生了冲突，优先用父类当中的方法

3.接口与接口之间多继承

格式

public interface MyInterface extends MyInterfaceA,MyInterfaceB{}

注意事项：
（1）多个父接口当中的抽象方法如果重复，没关系，即返回值类型相同
（2）多个父接口当中的默认方法如果重复，那么子接口必须进行默认方法的覆盖重写，而且带着default关键字

4.接口的默认方法

默认方法格式： 

public default 返回值类型 方法名称(参数列表){
    //方法体
}

实际代码：

public interface InterDefault {
    public abstract void methodAbs();
    public default void methodDefault(){
        System.out.println("默认方法！！！");
    }
}

接口当中的默认方法，可以解决接口升级的问题：
1.添加了新的方法，接口升级,部分接口可能已经投入使用，使用默认方法可以不需要大范围修改实现类也能实现新功能
2.能被直接调用的同时，也可以被重写

public class InterDefaultTest implements InterDefault{
    @Override
    public void methodAbs() {
        System.out.println("实现了！！");
    }

    @Override
    public void methodDefault() {
        System.out.println("重写默认方法！！");
    }
}

5.接口中成员变量

1.必须使用public static final三个关键字修饰，可以省略
2.从效果上看，这其实使接口的常量，一旦赋值不可以修改，定义时必须赋值
格式代码：

public interface InterConst {
    public static final int num =10;
}

6.接口抽象方法

1.抽象方法定义：

public abstract 返回值类型 方法名称(参数列表);

注意事项：
1.接口当中的抽象方法，修饰符必须是两个固定的关键字：public abstract
2.这两个关键字修饰符，可以选择性的省略(省略一部分或是全部）

public abstract void show();

7.接口静态方法

格式：

public static 返回值类型 方法名称(参数列表）{
      方法体
}

格式代码：

public static void methodStatic(){
    System.out.println("静态方法！！");
}

注意事项：
不能通过接口实现类对象来调用接口中的静态方法
正确用法：接口名.静态方法名();
