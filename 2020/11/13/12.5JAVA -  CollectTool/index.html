2020/11/12 JAVA学习笔记 集合方法&Map

一.处理集合的方法

相关包：java.utils.Collections
相关方法：

  • public static boolean addAll(Collection c,T… elements)向集合中添加一些元素
  • public static void shuffle(List<?> list):打乱集合顺序
  • public static void sort(List list):将集合中元素按照默认规则排序
  • public static void sort(List list,Comparator<? super T>)

1.addAll方法

Collections.addAll(list1,"a","b","c","d","e");

2.shuffle方法

Collections.shuffle(list1);

3.sort方法

将集合中元素按照制定规则排序,自定义类型需要重写compareTo进行排序
被排序的集合里边存储的元素必须实现Comparable,重写接口中的方法compareTo定义排序规则
步骤：

  1. 添加接口implements Comparable<类型>
  2. 重写compareTo方法

实例代码：
类中重写方法：

@Override
public int compareTo(Person o) {
    //默认返回0，认为元素都是相同的
    //自定义比较规则：
    return this.getAge() - o.getAge();//年龄升序排序，返回正值升序，返回负值降序
    //自己（this）-参数：升序
}

使用该方法：

Collections.sort(list2);

4.第二种sort方法

和第一种方法比较：

  • Comparable:this和参数比较，自己需要实现Comparable接口，重写比较规则compareTo方法
  • Comparator:相当于找一个第三方的裁判
    排序规则：
    o1-o2:升序，反之降序
    如果一样，则哪个写前面就在前面

实例代码：

Collections.sort(list2, new Comparator<Person>() {
        //重写
        @Override
        public int compare(Person o1, Person o2) {
            int result = o1.getAge() - o2.getAge();
            if (result == 0) {
                result = o1.getName().charAt(0) - o2.getName().charAt(0);
            }
            return result;
        }
    });

二.Map

1.定义

  • public interface Map<K,V>
  • java.util.Map<k,v>集合
  • 将键映射到值的对象，一个映射不能包含重复的键，每个键最多只能映射到一个值

2. 特点：

  • 双列集合，一个元素包含两个值(key,value)
  • key和value的数据类型可以相同，也可以不同
  • key不允许重复，value可重复
  • key和value一一对应

3.常用实现类

（1）java.util.HashMap<k,v>集合 implements Map<k,v>接口

HashMap基于哈希表的接口实现，此实现不同步，为多线程；无序

Map<String,String> map = new HashMap<>();

（2）java.util.linkedHashMap<k,v>集合 extends HashMap<k,v>集合

HashMap的子类LinkedHashMap介于哈希表和链表实现，具有可预知的迭代顺序，存储元素和取出元素的顺序是一致的
底层原理：哈希表加链表

4.常用方法

（1）boolean containsKey(Object key)判断集合中是否包含指定的键，有则true，无则false

System.out.println(map.containsKey("keyD"));

（2）public V get(Object key):根据指定的键，在Map集合中获取对应的值，没有则返回null

System.out.println(map.get("keyD"));

（3）public V remove(Object key):把指定的键所对应的键值对元素删除，返回被删除的值

String name = map.remove("keyE");
System.out.println(name);

注意事项：返回值尽量使用包装类，防止返回null时出现接收异常

（4）public V put(K key,V value):把指定的键与指定的值添加到Map集合中

key不重复返回值为null；重复的时候会使用新value值替换map中重复的value值，返回被替换的value值

String v1 = map.put("keyA","valueA");

4.遍历方法：

1.通过键找值的方式

方法Set keySet() 返回此映射中包含的键的Set视图
实现步骤：

  1. 使用Map集合中的方法keySet()，把Map集合所有的key取出来，存储到一个Set集合中
  2. 遍历set集合获取Map集合中的每一个key
  3. 通过Map集合中的方法get（key），通过key找到value

实例代码：

Set<String> set = map.keySet();
Iterator<String> it = set.iterator();
while(it.hasNext()){
    System.out.println(map.get(it.next()));
}

简化：

for(String key:map.keySet()){
    System.out.println(map.get(key));
}

2.使用Entry对象


Set<Map.Entry<K,V>> entrySet() 返回此映射中包含的映射关系的Set视图
实现步骤：

  1. 使用Map集合中的方法entrySet(),把Map集合中多个Entry对象取出来，存储到一个Set集合中
  2. 遍历Set集合，获取每一个Entry对象
  3. 使用Entry对象中的方法getKey()和getValue()获取键与值

实例代码：

Map<String,Integer> map = new HashMap<>();//使用包装类，不要使用基本类型
map.put("abc",132);
map.put("bvc",143);
map.put("pvc",145);

Set<Map.Entry<String,Integer>> set1 = map.entrySet();

Iterator<Map.Entry<String,Integer>> it1 = set1.iterator();
while(it1.hasNext()){
    Map.Entry<String,Integer> entry = it1.next();
    System.out.println(entry.getKey());
    System.out.println(entry.getValue());
}

for(Map.Entry<String,Integer> e: map.entrySet()){
    System.out.println(e.getKey() + " " + e.getValue());
}

5.保证元素唯一

HashMap存储自定义类型键值：
Map集合保证key是唯一的，作为key元素，必须重写hashCode方法和equals方法，以保证key唯一

6.补充知识点

HashTable
定义：java.util.Hashtable<K,V>集合 implements Map<K,V>接口
底层是一个哈希表，是一个线程安全的集合，是单线程集合，速度慢，被取代，不能存储null值null键
HashMap是一个线程不安全的集合，是多线程的集合，速度快，可以存储null值，null键
Properties集合是其子类，唯一一个和IO流相结合的集合
