2020/10/13 JS学习笔记

10.13 JS学习笔记

一.正则表达式

可以使用RegExp()构造函数来创建RegExp对象

var pattern = /s$/

1.定义：正则表达式直接量定义为包含在一对斜杠之间的字符,用来匹配所有以字母”s”结尾的字符串
运行这段代码创建一个RegExp对象，并将它赋值给变量pattern，等价于

var pattern1 = new RegExp("s$");

2.注意事项：
在ECMAScript3规范规定，一个正则表达式直接量会在执行到它时转换为一个RegExp对象，同一段代码所表示正则表达式直接量得每次运算都返回同一个对象；
但是，在ECMAScript5规范规定，同一段代码所表示的正则表达式直接量的每次运算都返回新对象，会导致运算结果的不同

二.直接量字符

var pattern2 = /\t/;    //制表符
var pattern3 = /\n/;    //换行符
var pattern4 = /\v/;    //垂直制表符
var pattern5 = /\f/;    //换页符
var pattern6 = /\r/;    //回车符
//var pattern7 = new RegExp("\xnn");  //由十六进制数nn指定的拉丁字符,显示报错
//var pattern8 = new RegExp("\uxxxx");    //由十六进制数xxxx指定的Unicode字符，例如\u0009等价于\t,报错
var pattern9 = /\cX/;    //控制字符^X,例如\cJ等价于换行符\n
var pattern10 = /\\/;    //用以匹配任何包含反斜线的字符串

三.字符类

1.定义：将直接量字符单独放进方括号内就组成了字符类

var pattern1A = /[abc]/;    //和字母"a""b""c"中的任意一个匹配

2.常用：
(1)”^”符号来定义否定字符类

var pattern1B = /[^abc]/;    //和字母“a""b""c"以外得任意字符匹配

(2)可以用连字符来表示字符范围

var pattern1C = /[a-zA-Z0-9]/;     //和a到z，A到Z以及0到9匹配

(3)其他常用字符类：

var pattern1D = /./;    //除换行符和其他unicode行终止符之外的任意字符
var pattern1E = /\w/;   //任何ASCII字符组成的单词，等价于[a-zA-Z0-9]
var pattern1F = /\W/;   //任何不适ASCII字符组成的单词，等价于[^a-zA-Z0-9]
var pattern1G = /\s/;   //任何Unicode空白符
var pattern1H = /\S/;   //任何非Unicode空白符的字符，注意\w和\S不同
var pattern1J = /\d/;   //任何ASCII数字，等价于p4[0-9]
var pattern1K = /\D/;   //除了ASCII字符之外的任何字符，等价于[^0-9]
var pattern1L = /[\b]/; //退格直接量

四.重复字符语法

{n,m} 匹配前一项至少n次，但不能超过m次
{n,} 匹配前一项n次以上
{n} 匹配前一项n次
？ 匹配前一次0次或1次，等价于{0，1}

  • 匹配前一项1次或多次，等价于{1，}
  • 匹配前一项0次或多次，等价于{0，}

重点：非贪婪的重复
在待匹配后的字符后跟随一个问号即可，”??”,”+?”，则会尽可能少地去进行匹配

/a+?/

可以匹配一个或多个，将”aaa”作为匹配字符串，只会匹配第一个a
注意事项：
使用/a+?b/匹配“aaab”会匹配整个字符，这是因为正则表达式的模式匹配总是会寻找字符串第一个可能匹配的位置，由于该匹配是从字符串地第一个字符开始，因此在这里不考虑他的子串更短的匹配

五.分组

var pattern2A = /ab|cd|ef/; //可以匹配字符串"ab""cd""ef",尝试匹配次序为从左到右

六.圆括号使用

作用：
1.把单独的项组合秤子表达式，以便可以像处理一个独立的单元那样用”|””*“”+””?”等对单元内的项处理，如/java(script)？/可以匹配”java”，其后可以由”script“也可以没有

2.在完整的模式中定义子模式
当一个正则表达式成功地和目标字符串相匹配时，可以从目标串中抽出和圆括号中的子模式相匹配的部分
例如/[a-z]+(\d)+/可以从检索到的匹配抽取数字

3.允许在同一正则表达式的后部引用前面的子表达式，这是通过在字符”"后加一位或多位数字来实现的。
这个数字指定了带圆括号的子表达式在正则表达式中位置,例如\1引用的是第一个带圆括号的子表达式

注意事项:
1.即使是嵌套的子表达式，也同样按从左到右计算左括号位置
2.：/[‘“][^’”]*[‘“]/可以用/[‘“][^’”]*\1/,不能写成/[‘“][^\1]*\1/
3.不适用创建带数字编码的引用，则以(?:和)来进行分组，且不及一与该组相匹配的字符

七.指定匹配位置

1.锚元素^,用以匹配字符串的开始,\b用以匹配单词的边界，即位于\w字符和\W之间的边界

2.先行断言
在符号(?=和）之间加入一个表达式，他就是一个先行断言，用以说明括号内的表达式必须正确匹配

3.负向先行断言
带有(?!的断言是负向先行断言，用以制定接下来的字符都不必匹配，和(?=均为零宽断言 P260

  • ^ 匹配字符串的开头，在多行检索中，匹配一行开头
  • $ 匹配字符串的结尾，在多行检索中，匹配一行的结尾
  • \b 匹配一个单词的边界，注意[\b]匹配的是退格符
  • \B 匹配非单词界面的位置

八.修饰符

定义：修饰符放在/符号外，也就是说，他们不是出现在两条斜线之间，而是第二条斜线之后

  • i 执行不区分大小写的匹配
  • g 执行一个全局匹配，即找到所有的匹配而不是在找到第一个之后就停止
  • m 多行匹配模式，则^会匹配一行的开头，$会匹配行的结束

九.使用正则表达式的方法

1.search()，它的参数是一个正则表达式，返回第一个与之匹配的子串起始位置，如果找不到则返回-1； 

"javaScript".search(/script/i); //返回4

如果参数部位正则表达式，则首先会通过RegExp构造函数将他转换成正则表达式，且不支持全局检索，会忽略修饰符g

2.replace()方法用以执行检索和替换操作，第一个参数为正则表达式，第二个为要进行替换的字符串；
正则表达式设置了修饰符g则源字符串中所有模式匹配的子串都将替换成第二个参数的字符串，不带g则只替换所匹配的第一个子串
注意事项：如果第一个是字符串则会直接搜索该字符串
例如：用于符号替换 

var quote = /"([^"]*)"/g;
//text.replace(quote,'“$1”');

3.match() 唯一参数为正则表达式，返回的是一个由匹配结果组成的数组；

"1 plus 2 equals 3".match(/\d+\g); //返回["1","2","3"]

不进行全局搜索时，只会检索第一个匹配，但仍然返回一个数组，数组的第一个元素是匹配的字符串，余下元素则是正则表达式用圆括号括起来的子表达式

4.split()方法

"123,456,789".split(",");   //返回["123","456","789"]
console.log("1,2,3,4,5".split(/\s*,\s*/));    //返回["1","2","3","4","5"]

十一.RegExp属性

  • source是一个只读字符串，包含正则表达式的文本
  • global是一个只读布尔值，说明正则表达式是否带有修饰符g
  • ignoreCase是一个只读布尔值，用以说明正则表达式是否带有i
  • multiline是一个只读布尔值，用以说明正则表达式是否带有m
  • lastIndex是一个可读/写整数，如果匹配模式带有g修饰符，这个属性存储在整个字符串中下一次检索的开始位置

RegExp方法:
1.exec()
对一个字符串执行一个正则表达式，匹配不到就返回null，匹配到了则返回数组同match()方法

2.test()
调用它和调用exec()等价，当exec()的返回结果不是null时，test()返回true
