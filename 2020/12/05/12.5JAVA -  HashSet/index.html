2020/11/12 JAVA学习笔记 Set类

一.Set类

1.定义：

java.util.Set接口 extends Collection接口

  • 1.一个没有重复元素的Collection
  • 2.没有索引

2.Set的两个实现类

（1）HashSet

java.util.HashSet类 implements Set接口

  • 不保证set的迭代顺序，不保证该顺序恒久不变，此类允许使用null元素
  • 不能重复元素、没有索引、无序、底层是一个哈希表结构（查询的速度非常的快）
哈希值：十进制整数，由系统给出，是对象模拟得出的逻辑地址值，不是数据实际存储的物理地址

在Object类有一个方法hashCode()返回对象的哈希码值
hashCode方法的源码：

public native int hashCode();
//native:代表该方法调用的是本地操作系统的方法
哈希表的结构：

Set集合不允许存储重复元素

实例代码：
1.非自定义类型： 

Set<Integer> hashSet1 = new HashSet<>();
hashSet1.add(1);
hashSet1.add(4);
hashSet1.add(3);
hashSet1.add(1);  //无用，不允许存储重复元素
Iterator<Integer> it = hashSet1.iterator();
while(it.hasNext()){
    System.out.println(it.next());
}
//使用增强for循环
for(Integer i:hashSet1){
    System.out.println(i);
}

String类的哈希值
//相关方法已经被重写
String s1 = new String("abc");
String s2 = new String("abc");
System.out.println(s1.hashCode());//96354
System.out.println(s2.hashCode());//96354
System.out.println(s1==s2);

2.自定义类型
需要重写对象中的hashCode和equals方法，建立自己的比较方式，才能保证HashSet结合中的对象唯一；
否则会存储编辑者认为的重复元素；

重写hashCode和equals的代码：
@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (o == null || getClass() != o.getClass()) return false;

    Person person = (Person) o;

    if (age != person.age) return false;
    return name != null ? name.equals(person.name) : person.name == null;
}

//hashCode在元素相同时相等，但是比较equals时返回true则无法存储相同元素
@Override
public int hashCode() {
    int result = name != null ? name.hashCode() : 0;
    result = 31 * result + age;
    return result;
}
实例代码：
Person p1 = new Person();
Person p2 = new Person();
   System.out.println(p1.hashCode());        //460141958
System.out.println(p2.hashCode());        //1163157884

System.out.println(p1);        //Person@1b6d3586  为460141958的16进制形式
System.out.println(p2);        //Person@4554617c
System.out.println(p1==p2);    //重写方法后返回的是false

（2）LinkedHashSet

java.util.LinkedHashSet集合 extends HashSet集合
特点：底层是一个哈希表（数组+链表/红黑树）+链表，多一条链表用于记录元素的存储数据，保证元素有序

实例代码：

LinkedHashSet<String> set1 = new LinkedHashSet<>();
set1.add("a");
set1.add("b");
set1.add("c");
set1.add("d");
System.out.println(set1);//有序，不允许重复

二.可变参数

1.使用前提

当方法的参数列表数据类型确定，个数不确定即可使用

2.使用格式：（定义方法时使用）

修饰符 返回值类型 方法名(数据类型... 变量){}

3.原理：

可变参数底层是一个数组，根据传递参数个数会创建不同长度的数组来存储参数

4.注意事项：

  • 一个方法参数列表只能有一个可变参数
  • 如果参数有多个，那么可变参数必须写在参数列表的末尾

特殊写法：数据类型为Object，可以接受任意数据类型的参数

实例使用代码：

private static void method(int a,int b,int... arr) {
    System.out.println(a);
    System.out.println(b);
    System.out.println(arr.length);
}

public static int sum(int... arr){
    int sum = 0;
    for(int i=0;i<arr.length;i++){
        sum+=arr[i];
    }
    return sum;
}
