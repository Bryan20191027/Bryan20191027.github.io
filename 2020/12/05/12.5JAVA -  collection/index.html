2020/11/12 JAVA学习笔记 Collection类

一.Collection类

1.定义

  • 数组长度固定，集合长度可变
  • 数组中存储的是同类型的元素，可以存储基本数据类型数据；
  • 集合存储的都是对象，而且对象的类型可以不一致

2.集合的框架

java.util.Collection接口

  • 所有单列集合的最顶层的接口，定义了所有单列集合共性的方法
  • 任意的单列集合都可以使用Collection接口中的方法

构造实例代码：

Collection<String> coll = new ArrayList<>();//可以改变子类类型，通用
System.out.println(coll);//输出[]

3.集合共性方法

  • public boolean add(E e):把给定的对象添加到当前集合中
  • public void clear():清空集合中所有的元素
  • public boolean remove(E e):把给定的对象在当前集合中删除
  • public boolean contains(E e):判断当前结合中是否包含给定对象
  • public boolean isEmpty():判断当前集合是否为空
  • public int size():返回集合中元素的个数
  • public Object[] toArray():把集合中的元素，存储到数组中(但不删除集合)

实例代码：

boolean b1 = coll.add("abc");//返回一个bool值，可以用于验证是否添加成功
coll.add("bcd");
coll.add("bve");
System.out.println(coll.size());
boolean b2 = coll.remove("be");
System.out.println(b2);

二.进行迭代

1.不使用迭代器

Object[] br = coll.toArray();
for(int i = 0;i<br.length;i++)
    System.out.println(br[i]);

2.使用迭代器Iterator

（1）迭代器接口

java.util.Iterator接口
对集合进行迭代的迭代器，是一个接口，无法直接使用；
需要使用Iterator接口的实现类对象，获取实现类的方式较为特殊
Collection接口中有一个方法iterator(),这个方法返回的就是迭代器的实现类对象

Iterator<E> iterator //返回在此collection的元素上进行迭代的迭代器

实例代码：

Iterator<String> it = coll.iterator();//Iterator接口也有泛型，迭代器的泛型跟着集合走

（2）常用方法

  • boolean hasNext()：如果仍有元素可以迭代，返回true
  • E next()：返回迭代的下一个元素
  • void remove():从迭代器指向的collection中移除迭代器返回的最后一个元素

（3）具体使用步骤

  1. 使用方法iterator（）获取迭代器实现类对象，使用Iterator接口接收（多态）
  2. 使用Iterator接口方法hasNext判断有无下一个元素
  3. 使用next()取出下一个元素

实例代码：

while(it.hasNext())
    System.out.println(it.next());
it.next();//没有元素，在取出元素时会抛出NoSuchElementException即没有元素异常

三.使用增强for循环进行遍历

增强for循环（foreach循环），内部是Iterator迭代器，所以在遍历过程中不能对集合中元素进行增删操作
集合中的迭代器接口允许集合对象成为foreach语句的目标
格式： 

for(集合/数组的数据类型 变量名：集合名/数组名){
      sout(变量名)；
}

实例代码：

//遍历数组
private static void demo01(){
    int[] arr = {1,2,3,4,5};
    for(int a:arr){
        System.out.println(a);
    }
}

//遍历集合
private static void demo02(){
    ArrayList<String> arr2 = new ArrayList<>();
    arr2.add("abc");
    arr2.add("bve");
    arr2.add("bcd");
    for(String str:arr2){
        System.out.println(str);
    }
}

四.泛型使用

1.是否使用泛型的比较

不使用泛型：

  • 好处：集合不使用泛型，默认类型为Object类型，可以存储任意类型的数据
  • 弊端：不安全，会发生异常
    例如迭代向下转型的时候，把Integer类型转换为String类型，抛出ClassCastException类型转换异常

使用泛型：

  • 好处：1.避免了类型转换的麻烦 2.把运行期异常提升到了编译时异常
  • 弊端：泛型是什么类型，数据只能存储什么类型

2.泛型的定义

泛型是一个未知的数据类型，当我们不知道使用什么类型的时候，就可以使用泛型
创建对象的时候可以确定是什么类型，或是不写默认为Object类型

（1）含有泛型的类

实例代码：
public class newGeneCls {
private E name;

    public E getName() {
        return name;
    }

    public void setName(E name) {
        this.name = name;
    }
}

（2）含有泛型的方法

格式:

修饰符<泛型> 返回值类型 方法名(参数列表(使用泛型））{
    //方法体
}

注意事项：

  • 含有泛型的方法在调用方法的时候确定泛型的数据类型
  • 传递什么类型泛型就是什么类型

实例代码：

public static<M> void show(M m){
    System.out.println(m);
}

（3）含有泛型的接口

使用方法：

  1. 定义接口的实现类，实现接口，指定接口的泛型
  2. 接口使用什么类型的泛型，实现类就使用什么类型的泛型

实例代码：

public interface genericsInter<I> {
    public abstract void show(I i);
}//含有泛型的接口

public class newGeneCls1<E> implements genericsInter<E>{
    @Override
    public void show(E e) {
        System.out.println(e);
    }
}//定义未指定泛型的接口实现类

public class geneInterImp implements genericsInter<String>{
       @Override
    public void show(String s) {
        System.out.println(s);
    }
}//泛型接口的实现类，指定泛型为String

五.泛型通配符

1.定义：

当使用泛型类或接口时，数据类型不确定，可以通过通配符<?>表示

注意事项：

一旦使用泛型通配符后，只能使用Object类中的共性方法，集合中元素自身方法无法使用

基本使用：

？表示未知通配符，任意的数据类型
使用方式：不能创建对象使用，只能作为方法的参数使用

实例代码：

public static void show1(ArrayList<?> coll){
    Iterator<?> it = coll.iterator();
    while(it.hasNext()){
        Object obj = it.next();//用Object类型接受任意类型的数据
        System.out.println(obj);
    }
}

2.通配符高级使用——受限泛型

上限格式：类型名称<? extends 类> 对象名称
  意义：只能接受该类型及其子类
下限格式：类型名称<? super 类> 对象名称
  意义：只能接受该类型及其父类型

六.list接口

1.特点：有序、可重复、有索引

java.util.List接口 extends Collection接口

2.接口方法：

  • public void add(int index,E element):添加元素到指定位置
  • public E get(int index):返回集合中指定位置的元素,若索引越界会出现IndexOutOfBoundsException异常
  • public E remove(int index)：移除列表指定位置元素，返回被移除元素
  • public E set(int index,E element):用指定元素替换指定位置元素，返回更新前元素

实例代码：

List<String> list = new ArrayList<>();//多态
list.add("a");//使用collection接口add方法
list.add("b");
list.add("c");
list.add("d");
list.add("e");
list.add(3,"itheima");//index为需要添加的位置
String e = list.remove(2);
String setE = list.set(3,"aaa");

3.接口实现类

（1）ArrayList，底层是一个数组结构

（2）LinkedList集合，List接口的链表实现，和ArrayList一样实现不是同步的(多线程的)

  • 底层是链表结构
  • 包含了大量操作首尾元素的方法
  • 注意：如果需要使用LinkedList集合特有的方法，则不能使用多态

LinkedList使用方法：
1.添加方法

  • public void addFirst(E e):将指定元素插入此列表的开头
  • public void addLast(E e):将指定元素添加到此列表的结尾
  • public void push(E e):将元素推入此列表所表示的堆栈

实例代码：

linked.addFirst("www");
//linked.push("www");  效果一样
linked.addLast("wwLast");

2.删除方法

  • public E removeFirst():移除并返回此列表的第一个元素
  • public E removeLast():移除并返回此列表的最后一个元素
  • public E pop():从此列表所表示的堆栈处弹出一个元素

实例代码：

String a = linked.removeFirst();
//相当于linked.pop();
System.out.println(linked.removeLast());//返回最后一个元素

3.获取方法

  • public E getFirst():返回列表的第一个元素
  • public E getLast():返回此列表的最后一个元素

实例代码：

System.out.println(linked.getLast());
System.out.println(linked.getFirst());

4.判断列表是否为空

public boolean isEmpty():如果列表不包含元素，返回true
