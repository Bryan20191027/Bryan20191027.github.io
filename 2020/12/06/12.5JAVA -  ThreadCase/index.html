2020/11/13 JAVA学习笔记 Thread案例

一.等待唤醒案例：线程之间的通信

创建消费者：告知所需，调用wait方法，放弃cpu执行，进入WAITING状态

 new Thread(){
        @Override
        public void run(){
            //保证等待和唤醒线程只有一个执行，使用同步
            synchronized (obj){
                System.out.println("告知需要");
                try {
                    obj.wait();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                //唤醒之后执行代码
                System.out.println("接取所需");
            }
        }
    }.start();

创建生产者：完成所需，后调用notify方法，唤醒消费者

new Thread(){
        @Override
        public void run(){
            try {
                Thread.sleep(5000);//写在外部可以保证消费者线程先执行
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            synchronized (obj){
                System.out.println("5秒后完成");
                obj.notify();
            }
        }
    }.start();

注意：消费者和生产者必须使用同步代码块包裹起来，保证等待和唤醒只能有一个执行
同步使用的锁对象必须保证唯一

//创建锁对象，保证唯一
Object obj = new Object();

只有锁对象才能调用wait和notify方法

共同处理一个资源型（类）：

Object类方法

  • void wait()：在其他线程调用此对象的notify()方法或notifyAll()方法前，导致当前线程等待
  • void notify:唤醒在此对象监视器上等待的单个线程，会继续执行wait方法后代码

TimeWaiting两种方式：

  • sleep(long m),在结束后线程进入到Runnable/Blocked状态
  • wait(long m)方法，wait方法如果在毫秒值结束之后，还没有被notify唤醒就会自动醒来

唤醒方法：

  • notify()
  • notifyAll():唤醒所有等待线程

1.同步代码块

格式：

synchronized(锁对象){
    可能会出现线程安全问题的代码(访问了共享数据的代码)
}

注意：
1.同步代码块中的锁对象，可以使用任意的对象，但是必须保证多个线程使用的锁对象是同一个
2.锁对象作用：把同步代码块锁着，只让一个线程在同步代码块中执行

实例代码：

while(true){
        //同步代码块
        synchronized (obj){
            if(ticket>0){

                    e.printStackTrace();
                }
                System.out.println(Thread.currentThread().getName() + "--->正在卖第" + ticket + "张票");
                ticket--;
            }
        }
    }

2.同步方法

定义：保证一个线程执行该方法的时候，其他线程只能在方法外等着

格式：public synchronized void method(){}
使用步骤：

  1. 把访问了共享数据的代码抽取出来，放到一个方法中
  2. 在方法上添加synchronized修饰符

格式：定义方法的格式
修饰符 synchronized 返回值类型 方法名(参数列表){}；
定义一个同步方法，会把方法内部代码锁住，只让一个线程执行，锁对象为实现类对象RunnableImpl(),也就是this

实例代码：

public static synchronized void payTicketStatic(){
    synchronized (RunnableImpl1.class) {
        if (ticket > 0) {
            try {
                Thread.sleep(10);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(Thread.currentThread().getName() + "--->正在卖第" + ticket + "张票");
            ticket--;
        }
    }
}

3.锁机制

lock锁：java.util.concurrent.locks.Lock接口
Lock实现提供了比使用synchronized方法和语句可获得的更广泛的锁定操作
Lock接口中的方法:

  • void lock()获取锁
  • void unlock()释放锁
  • java.util.concurrent.locks.ReentrantLock implements Lock接口

使用步骤：

  1. 在成员位置创建一个ReentrantLock对象
  2. 在可能会出现安全问题的代码前调用Lock接口中的方法lock获取锁
  3. 在可能会出现安全问题的代码后调用Lock接口中的方法unlock释放锁

实例代码：

public class RunnableImpl2 implements Runnable{
private int ticket = 100;

//1.在成员位置创建一个ReentrantLock对象
Lock l = new ReentrantLock();

@Override
public void run() {
    while(ticket > 0){
        //2.lock获取锁
        l.lock();
        if (ticket > 0) {
            try {
                Thread.sleep(10);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(Thread.currentThread().getName() + "--->正在卖第" + ticket + "张票");
            ticket--;
        }
        //3.unlock释放锁
        l.unlock();
    }
}

}

5.线程池

定义：一个容纳多个线程的容器，其中线程可以反复使用，省去了频繁创建线程对象的操作，无需反复创建线程而消耗过多资源
好处：

  • 降低资源消耗
  • 提高响应速度
  • 提高线程的可管理型

原理图：

（1）线程池的使用

java.util.concurrent.Executors:线程池的工厂类，用来生成线程池
Executors类中的静态方法：
static ExecutorService newFixedThreadPool(int nThreads)创建一个可重用固定线程数的线程池
参数：int nThreads:创建线程池中包含的线程数量
返回值：ExecutorService接口，返回是的ExecutorService接口的实现类对象，可以使用ExecutorService接口接收

（2）线程池接口

java.util.concurrent.ExecutorService
用来从线程池中获取线程，调用start方法，执行线程任务

  • submit(Runnable task) 提交一个Runnable任务用于执行
  • void shutdown() 关闭/销毁线程池的方法

使用步骤：

  1. 使用Executors里边提供的静态方法newFixedThreadPool生成一个指定线程数量的线程池
  2. 创建一个类，实现Runnable接口重写run方法，提供线程任务
  3. 调用ExecutorService中方法submit，传递线程任务开启线程
  4. 调用shutdown销毁线程池(不建议执行)

实例代码：

ExecutorService es = Executors.newFixedThreadPool(2);
es.submit(new runnableCls());
//线程池会一直开启，使用完线程，会自动把先后曾归还给线程池，线程可以继续使用
es.submit(new runnableCls());
es.submit(new runnableCls());
//es.shutdown();
