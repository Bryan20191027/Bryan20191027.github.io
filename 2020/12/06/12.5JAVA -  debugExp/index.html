2020/11/13 JAVA学习笔记 debug&&Exception

一.debug

可以让代码逐行执行，查看代码执行的过程，调试程序中出现的bug

1.使用方式：

在行号右边鼠标左键单击，添加断点（每个方法的第一行，那里有bug添加到哪里）；
右键，选择Debug执行程序，程序就会停留在添加的第一个断点

2.执行方式：

  • f8:逐行执行程序
  • f7:进入到方法中
  • shift+f8：跳出方法
  • f9:跳到下一个断点，如果没有下一个断点，那么久结束程序
  • ctrl+f2:推出到debug模式，停止程序
  • Console:切换到控制台

二.异常（重点）

1.定义： 指的是程序在执行过程中，出现的非正常的情况，最终会导致JVM的非正常停止

  • 在java等面向对象的编程语言中，异常本身是一个类，
  • 产生异常就是创建异常对象并抛出了一个异常对象，Java处理异常的方式是中断处理

2.异常体系

根类是java.lang.Throwable(是java语言中所有错误或异常的超类),其下两个子类：

  • java.lang.error
  • java.lang.Exception

平常说的异常指的是java.lang.Exception
1.Exception:编译器异常，进行编译java程序出现的问题
 RuntimeException：运行期异常，java程序运行过程中出的问题
2.Error:错误，必须修改源代码才能继续执行 

int[] arr = new int[1024*1024*1024];//OutOfMemory错误，内存溢出错误，必须修改源代码

3.处理方法

  1. 抛出，交给虚拟机处理
  2. try—catch，后面代码可以继续执行

4.异常的产生过程解析

5.具体处理

（1）throw关键字

作用：可以使用throw关键字在指定的方法中抛出指定的异常
使用格式：

throw new xxxException("异常产生的原因")

注意：

  1. throw关键字必须写在方法的内部
  2. throw关键字后边new的对象必须是Exception或者其子类对象
  3. throw关键字抛出指定的异常对象，必须处理该对象

如果throw关键字后创建的是RuntimeException或者是其子类对象，我们可以不处理，默认交给JVM处理（打印异常对象，中断程序）
throw后创建的是编译异常(写代码时候报错)，必须处理该对象，要么throws，要么try…catch
实例代码：

if(arr == null){
    throw new NullPointerException("传递数组的值是null");
}
if(index < 0||index > arr.length -1){
    throw new ArrayIndexOutOfBoundsException("传递索引超过了数组的适用范围");
}
int ele = arr[index];
return ele;

（2）相关异常判断

  • 对方法传递过来的参数进行合法性校验，若参数不合法，那么我们就必须使用抛出异常的方式警告传递参数有问题
  • Objects非空判断，查看指定引用对象是不是null

相关方法：

public static <T> T requireNonNull(T obj):查看指定引用对象不是null

源码：

public static <T> T requireNonNull(T obj){
    if(obj == null)
        throw new NullPointerException();
    return obj;}

（3）throws关键字：异常处理的第一种方式，交给别人处理

作用：当方法内部抛出异常对象的时候，我们就必须处理这个异常对象
可以使用throws关键字处理异常对象，会把异常对象生命抛出给方法的调用者处理，最终交给JVM处理–》中断处理

使用格式：在方法声明时使用
修饰符 返回值类型 方法名(参数列表) throws AAAException,BBBException...{
    throw new AAAException("...");
    throw new BBBException("...")
}

注意：

  1. throws关键字必须写在方法声明处
  2. throws关键字后边声明的异常必须是Exception或其子类
  3. 方法内部如果抛出多个异常对象，则throws后也必须声明多个异常，如果抛出多个异常对象有子父类关系，那么直接声明父类异常即可
  4. 调用了一个声明抛出异常的方法，我们就必须处理生命的异常，要么继续使用throws声明抛出直至交给JVM，要么try…catch自己处理

实例代码：

public static void main(String[] args) throws IOException {
    readFile("c:\\a.tx");
}
//定义方法，对传递文件路径进行合法性判断
//注意：FileNotFound是编译异常，抛出了编译异常,就必须处理
//可以使用throws继续声明抛出FileNotFoundException这个异常对象，让方法调用者处理
public static void readFile(String fileName) throws IOException{
    if(!fileName.equals("c:\\a.txt")){
        throw new FileNotFoundException("传递文件路径不是c:\\a.txt");
    };
    //如果不是.txt结尾，就抛出IO异常对象，告知方法调用者，文件后缀名错误
    if(fileName.endsWith(".txt")){
        throw new IOException("后缀名不对");
    }
    System.out.println("路径没有问题");
}

（4）try…catch代码块

格式：

try{
      可能产生异常的代码;
}catch(定义一个异常的变量，用来接收try中抛出的异常对象){异常的处理逻辑，怎么处理异常对象
     (工作中会记录到日志中)
 }
...
catch(异常类名 变量名){}

注意事项：

  1. try中可能会抛出多个异常对象，那么就可以使用多个catch来处理这些异常对象
  2. 如果try中出现异常，呢么就会执行catch中的异常处理逻辑，执行完catch中的处理逻辑继续执行try…catch后代码；如果try中没有产生异常，那么就不会执行catch中异常的处理逻辑，执行完try中的代码继续执行try…catch后的代码

实例代码：

try {
        readFile("c:\\b.txt");
    } catch (IOException e) {
        System.out.println(e.getMessage());
        System.out.println(e.toString());
        e.printStackTrace();
        System.out.println("catch-传递后缀名不是.txt");
    }
补充：异常查看方法
  • public String getMessage():获取异常描述信息，原因,返回简短描述
  • public String toString():获取异常类型和异常描述信息，返回详细消息字符串
  • public void printStackTrace():打印异常的跟踪栈信息并输出到控制台，JVM打印异常对象，默认方法

（5）finally代码块

接在catch(){}后面，格式：

finally{无论是否出现异常都会执行}

注意：

  • 1.finally不能够单独使用，必须和try使用
  • 2.finally一般用于资源释放，无论程序是否出现异常，最后都要资源释放(IO)

实例代码：

int a = 10;
try{
    return a;
}catch (Exception e){
    System.out.println(e);
}
finally {
    a = 100;
    return a;
}

（6）多个异常捕获

  • 运行时异常被抛出可以不处理，即不捕获也不声明抛出，默认给虚拟机处理，终止程序
  • 如果finally有return语句，则永远返回finally中的结果，避免该情况,尽量不要在finally写return
    如果父类抛出了多个异常，子类重写父类方法是，抛出和父类相同的异常或者是父类异常的子类或者不抛出异常
    父类方法没有抛出异常，子类重写父类该方法时也不可抛出异常，此时子类产生该异常，只能捕获处理，不能声明抛出

实例代码：

 /*多个异常使用捕获
  1.多个异常分别处理
  2.多个异常一次捕获，多次处理
  3.多个异常一次捕获一次处理
 */
 //1
 int[] arr = {1,2,3};
 try{System.out.println(arr[3]);}
 catch(ArrayIndexOutOfBoundsException a){
     System.out.println(a);
 }

 List<Integer> list = new ArrayList<>();
 list.add(1);
 list.add(2);
 list.add(3);
 try{System.out.println(list.get(3));}
 catch (IndexOutOfBoundsException e){
     System.out.println(e);
 }

 //2.
 try{
     System.out.println(arr[3]);
     System.out.println(list.get(3));//不会运行
 }catch(ArrayIndexOutOfBoundsException a){
     System.out.println(a);
 }catch (IndexOutOfBoundsException e){
     System.out.println(e);
 }
 //注意：catch里面定义的多个异常变量，如果有子父类关系，那么子类的异常变量必须写在上边
 //否则报错，ArrayIndexOfOutOfBoundsException extends IndexOutOfBoundsException

 //3.
 try{
     System.out.println(arr[3]);
     System.out.println(list.get(3));//不会运行
 }catch (IndexOutOfBoundsException e){
     System.out.println(e);
 }

（7）自定义异常类

java提供的异常类可能不够我们使用，需要自己定义一些异常类
格式：

public class xxxException extends Exception ||RuntimeException(){
     添加一个空参数构造方法，一个带异常信息的构造方法；
 }

自定义异常类，必须继承Exception或者RuntimeException

  • 继承Exception：那么自定义的异常类就是一个编译器异常
  • 继承RuntimeException:那么自定义异常类就是一个运行时异常(中断处理)

实例代码：

public class RegException extends Exception{
    //查看源码发现，所有的异常类都会有一个带异常信息的构造方法，
    // 方法内部会调用父类带异常信息的构造方法，让父类来处理这个异常信息
    public RegException() {
        super();
    }
    public RegException(String message){
        super(message);
    }
}
